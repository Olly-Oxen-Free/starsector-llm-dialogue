# Starlogue Handoff (2026-04-23)

This document is for handing the current Starlogue work to another AI/agent with minimal re-discovery.

## Current Goal Context

Recent work focused on:

- Fixing Starlogue option injection behavior (bar/comm vs normal encounter dialogs)
- Improving LLM backend reliability (multi-backend ordered failover)
- Adding xAI support
- Fixing composer behavior (long input + wrapping)
- Blocking non-conversable targets (derelicts/asteroid-like clutter)
- Handling invalid model temperature errors more gracefully

## Runtime State Observed In Logs

Source: `starsector.log`.

Key runtime failures currently seen:

- OpenRouter primary fails with `HTTP 402` (insufficient credits / account has not purchased credits).
- Kimi direct fallback fails with `HTTP 401` (incorrect or missing API key).
- Model-specific error observed for Kimi: `HTTP 400 invalid temperature: only 1 is allowed for this model`.
- Audit logger shows periodic write failures (`ConversationAuditLog` warnings), but this does not block core UI flow.

## Code Changes Already In Place

### 1) Ordered backend array failover

Implemented in:

- `src/starlogue/config/LlmBackendConfig.java`
- `src/starlogue/ui/StarlogueDialogPlugin.java`

Behavior:

- Supports `starlogue_backends` array in credentials JSON.
- Attempts backend entries in order: index 0, then 1, then 2...
- Keeps legacy single-backend fields for backward compatibility.
- OpenRouter-specific retries still apply per backend attempt:
  - retry without tools when provider reports no tool endpoint support
  - retry with `openrouter/auto` on provider errors

### 2) xAI provider support

Implemented in:

- `src/starlogue/llm/XaiClient.java`
- `src/starlogue/config/LlmBackendConfig.java`
- `src/starlogue/ui/StarlogueDialogPlugin.java`

Behavior:

- Provider aliases normalize to `xai` (`x.ai`, `grok`).
- Default endpoint: `https://api.x.ai/v1`.
- Integrated in config validation and client routing.

### 3) Invalid temperature auto-retry fallback

Implemented in:

- `src/starlogue/ui/StarlogueDialogPlugin.java`

Behavior:

- If request fails with a temperature-related invalid value message (e.g. "only 1 is allowed"),
Starlogue retries once with fallback temperature `1.0` for that backend/model.

### 4) Broken duplicate option injection fix

Implemented in:

- `src/starlogue/ui/StarlogueOptionEnforcerScript.java`

Behavior:

- Enforcer now skips `RuleBasedDialog` screens (where `rules.csv` injection already works).
- Enforcer now limits itself to non-rule bar/comm-like dialogs.
- Prevents extra broken "Hail/Open comm link" option from appearing in normal fleet encounter menus.

### 5) Non-conversable target blocking

Implemented in:

- `src/starlogue/engine/StarlogueTargetFilter.java`
- `src/starlogue/engine/ConstraintEngine.java`

Behavior:

- Excludes salvage/derelict/terrain/asteroid-like targets (incl. custom type heuristics).
- Stops catch-all `SystemAIPlugin` from opening on junk entities.

### 6) Person recovery for bar/comm dialogs

Implemented in:

- `src/starlogue/ui/RuleMemoryHelper.java`
- `src/starlogue/provider/MarketAdminPlugin.java`

Behavior:

- Better `$person` recovery from dialog/plugin/nested fields and alternate memory keys.
- Avoids false early returns when plugin memory map is empty.

### 7) Composer wrap behavior

Implemented in:

- `src/starlogue/ui/StarlogueDialogPlugin.java`

Behavior:

- Long text remains allowed (`setMaxChars(200000)`).
- Width-limit mode is ON to restore multiline wrapping.
